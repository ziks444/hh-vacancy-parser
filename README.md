# 🕷️ HH.ru Vacancy Parser — Парсер вакансий Python-разработчиков

Автоматизированный инструмент на Python для сбора и анализа актуальных вакансий Python-разработчиков с сайта hh.ru. Парсер использует библиотеку **Selenium** для работы с динамическим контентом, обрабатывает данные (конвертация валют, нормализация зарплат) и сохраняет результаты в Excel-файл с автоматически построенными графиками.

Проект разработан в рамках учебной практики по анализу рынка труда IT-специалистов.

## 📊 Функциональность

- ✅ Автоматический сбор вакансий по заданным параметрам (профессия, регион)
- ✅ Фильтрация только релевантных Python-вакансий
- ✅ Извлечение полной информации: название, зарплата, опыт, навыки, график работы
- ✅ Конвертация валют (USD, EUR → RUB) по заданным курсам
- ✅ Приведение зарплат к месячному эквиваленту (часовые, дневные, недельные ставки)
- ✅ Сохранение результатов в Excel с 5 листами
- ✅ Автоматическое построение 4 типов графиков (гистограмма, столбчатая, горизонтальная, круговая диаграммы)
- ✅ Обход базовой защиты от ботов

## 🛠️ Технологии

- **Python 3.10+**
- **Selenium** — управление браузером Chrome, работа с динамическим контентом
- **pandas** — обработка и структурирование данных
- **xlsxwriter** — создание Excel-файлов с форматированием и встроенными графиками
- **matplotlib** — построение графиков и диаграмм
- **re** — регулярные выражения для парсинга зарплат
- **collections.Counter** — подсчёт частоты навыков

## 🚀 Запуск

```bash

```

Парсер автоматически:
1. Откроет браузер Chrome
2. Пройдёт по 10 страницам поиска hh.ru
3. Соберёт ссылки на Python-вакансии
4. Запарсит детали каждой вакансии
5. Обработает зарплаты (конвертация валют, приведение к месячному формату)
6. Сохранит результаты в файл `vacancies_moscow.xlsx`

## ⚙️ Как работает парсер

### Архитектура

Парсер состоит из нескольких функциональных модулей:

1. **Модуль инициализации браузера** — настраивает Chrome для обхода защиты от ботов (отключение флагов автоматизации, кастомный user-agent)
2. **Модуль сбора ссылок** — проходит по страницам поиска и извлекает URL вакансий по CSS-селектору `a[data-qa='serp-item__title']`
3. **Модуль парсинга деталей** — заходит на каждую страницу и собирает данные из элементов с атрибутами `data-qa`
4. **Модуль обработки зарплаты** — конвертирует валюты и приводит к месячному эквиваленту
5. **Модуль фильтрации** — оставляет только полные вакансии с Python-стеком
6. **Модуль экспорта** — сохраняет данные в Excel и строит графики через matplotlib

### Ключевые особенности

- **Selenium вместо API** — работа с динамическим контентом, загружаемым через JavaScript
- **data-qa селекторы** — использование стабильных атрибутов hh.ru, предназначенных для автотестов
- **Обход защиты** — отключение флагов автоматизации, кастомный user-agent, случайные задержки между запросами
- **Обработка исключений** — парсер не падает при отсутствии полей в вакансии
- **Валидация данных** — сбор только полных вакансий для корректной статистики

### Обработка зарплаты

Парсер распознаёт:
- **Период**: за час, день, неделю, месяц, проект (по ключевым словам)
- **Валюту**: RUB, USD, EUR (по символам `$`, `€`, `₽` и текстовым маркерам)
- **Конвертация**: 1 USD = 95 RUB, 1 EUR = 103 RUB
- **Приведение к месячному**: часовая × 168, дневная × 22, недельная × 4.3

**Пример**: `"от 5 000 $ за месяц"` → 5000 × 95 = **475 000 руб./мес.**

## 🔧 Настройка

В начале файла `parser.py` можно изменить основные параметры:

```python
# Поисковый запрос
query = "Python Developer"

# Количество вакансий для сбора
target_count = 10

# Количество страниц для обхода
pages = 10

# Курсы валют
USD_TO_RUB = 95.0
EUR_TO_RUB = 103.0

# Коэффициенты для месячного эквивалента
HOURS_PER_MONTH = 168
DAYS_PER_MONTH = 22
WEEKS_PER_MONTH = 4.3
```

Для поиска других профессий достаточно изменить переменную `query`, например:
```python
query = "Java Developer"
query = "Data Analyst"
query = "Frontend Developer"
```
